# qifan 本地知识库

这是一个纯静态的个人知识库网页，用于集中浏览读书知识、饭局社交攻略、情绪沟通课程和《资治通鉴》解读等内容。

项目目标不是做成营销型首页，而是做成长期可用的阅读型知识产品：桌面端强调查找效率，手机端强调顺滑阅读，折叠屏和平板强调左右分工。

## 设计原则

- 结构清晰：桌面端采用筛选、列表、详情三栏；中等屏采用双栏；手机端采用单列流程。
- 色彩克制：整体控制在背景色、面板色、文字色、主色四类颜色内，蓝色只用于当前选中和主操作。
- 字体稳定：中文优先使用系统字体，正文保持舒适行高，避免默认小灰字带来的廉价感。
- 留白统一：间距遵循 4 / 8 / 12 / 16 / 24 / 32 / 48 的体系，避免随意魔法数字。
- 动效克制：只保留 hover、选中态和手机端平滑滚动，不做炫技动画。

## 目录结构

- `index.html`：页面骨架，只保留结构和资源引用。
- `assets/css/styles.css`：样式入口文件，只负责引入各层 CSS。
- `assets/css/tokens.css`：颜色、字体、间距、圆角、阴影等设计令牌。
- `assets/css/base.css`：全局基础样式和滚动条样式。
- `assets/css/layout.css`：页面布局、三栏结构和阅读区约束。
- `assets/css/components.css`：搜索框、筛选按钮、卡片、标签、详情、表格等组件样式。
- `assets/css/responsive.css`：桌面、折叠屏/平板、手机、小手机的响应式规则。
- `assets/js/data.js`：生成后的知识库数据，向页面暴露 `window.KNOWLEDGE_DATA`。
- `assets/js/app.js`：搜索、筛选、卡片列表、详情渲染和手机端交互。

## 维护约定

- 新增或更新知识库内容时，优先更新 `assets/js/data.js`。
- 调整颜色、间距、字号时，优先更新 `assets/css/tokens.css`。
- 调整桌面、折叠屏、手机端布局时，优先更新 `assets/css/responsive.css` 和 `assets/css/layout.css`。
- 新增筛选、排序、收藏等交互时，优先更新 `assets/js/app.js`。
- 不要把长数据、复杂样式或大段脚本重新写回 `index.html`。
